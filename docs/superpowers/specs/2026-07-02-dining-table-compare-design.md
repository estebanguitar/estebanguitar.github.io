# 식탁 비교 페이지 설계서 (table-compare.html)

작성일: 2026-07-02
대상: `/Users/joseph/interior-compare/table-compare.html` (신규)

## 목표

4인 기준 **식탁 100개**를 수집하는 단일 비교 페이지를 만든다. 침대 페이지(`bed-compare.html`)와 **동일한 방식**(정적 데이터 + 상세 URL 전수 검증 + 이미지 전수 검증 + 브랜드별 커밋/푸시)으로 구축한다.

핵심 컨셉: **식탁 중심 페이지**. 식탁의자는 식탁에 **종속**된 속성으로 다룬다. 각 식탁 카드에 매칭되는 식탁의자의 **개당 가격**을 표기하고, **식탁 + 의자 4개 합계 가격**을 함께 보여준다.

## 확정된 요구사항

| 항목 | 결정 |
|---|---|
| 페이지 구조 | 단일 페이지 `table-compare.html`, 식탁 중심 (의자 종속) |
| 수집 개수 | 식탁 **100개** (각각 매칭 의자 포함) |
| 식탁 사이즈 기준 | 4인 = 폭 1200~1500mm × 깊이 750~900mm → `ok` |
| 예산 기준 | **식탁 가격 + 의자 개당가격 × 4 ≤ 2,000,000원** |
| 예산 초과 상품 | 목록에 포함하되 `예산 초과` badge 표시 (제외 안 함) |
| 의자 검증 수준 | 실존 상품 **이름 + 개당가격만** 기록. 의자 URL/이미지는 검증하지 않음 |
| 식탁 본체 검증 | 침대와 동일 — 실제 상세 URL(홈/검색/카테고리 폴백 0), 이미지 HTTP 200·`image/*`·해상도·중복 검증 |
| Git | 브랜드별 1커밋 → `git push origin main`, 한 번에 한 브랜드 |

## 데이터 구조

`table-compare.html`의 `<script>`에 정적 배열 `const TABLES=[...]`로 저장한다. 각 원소:

```javascript
{
  brand:"브랜드명",
  name:"식탁 제품명 전체",
  status:"ok|near|check|exceed|ref",
  material:"wood|ceramic|glass|marble|veneer|metal|unknown", // 상판 소재
  shape:"rect|round|oval|ext",       // 직사각/원형/타원/확장형
  seat:"4|4-6",                       // 인원 기준
  price:890000,                       // 식탁 단품 가격
  orig:null,                          // 정가(없으면 null)
  url:"https://식탁-상세페이지-URL",   // 실제 상세페이지 (검증 대상)
  img:"https://식탁-대표이미지-URL",   // 실제 이미지 (검증 대상)
  chairName:"매칭 식탁의자 제품명",     // 실존 의자
  chairPrice:120000,                  // 의자 개당 가격
  setTotal:1370000,                   // = price + chairPrice*4 (표시/정렬용, 정적 계산해 저장)
  badge:"4인 세트 확인",
  bc:"b-ok",
  specs:[
    ["상판","세라믹 1400×800mm","ok"],
    ["사이즈","4인 폭1400×깊이800mm ✓","ok"],
    ["형태","직사각 확장형",""],
    ["다리","원목 4다리",""],
    ["의자","패브릭 의자 4개 포함",""]
  ]
}
```

### status / badge 규칙 (침대 규칙 준용)

- `ok` + `b-ok`: 4인 사이즈 확인 + 세트 ≤ 200만 → badge `"4인 세트 확인"`
- `near` + `b-near`: 사이즈 준하거나 세트가 200만 근접 → badge `"예산 근접"`
- `check` + `b-warn`: 사이즈/구성 확인 필요 → badge `"사이즈 확인 필요"`
- `exceed` + `b-fail`: 세트 > 200만 → badge `"예산 초과"`
- `ref` + `b-ref`: 단종/참고 → badge `"단종 가능 — 참고"`

## 카드 UI

각 카드에 세 가격을 명시한다:

```
[식탁 대표 이미지]
브랜드 · 식탁 제품명
식탁            890,000원
의자(개당)      120,000원 ×4
──────────────────────────
세트 합계     1,370,000원   [4인 세트 확인]
```

## 필터 / 정렬

- **상판 소재**: 전체 / 원목 / 세라믹 / 강화유리 / 대리석 / 무늬목·MDF / 철제
- **형태**: 전체 / 직사각 / 원형 / 타원 / 확장형
- **인원**: 전체 / 4인 / 4~6인
- **예산**: 전체 / 세트 200만 이하만
- **정렬**: 기본순 / 세트 합계 낮은순 / 세트 합계 높은순

## 네비게이션 / 진입점

- `nav.js` 공유 컴포넌트에 탭 추가: `🍽 식탁 비교` (`/table-compare.html`)
- `index.html`에 식탁 비교 링크 추가
- 기존 3개 페이지(sofa/chair/bed)와 동일한 스타일·레이아웃 재사용 (`bed-compare.html`을 기준 템플릿으로)

## 데이터 수집 · 검증 절차 (침대와 동일)

1. 브랜드/판매처별로 4인 식탁 실존 상품 조사 (공식몰·다나와 개별상품·네이버 catalog 등)
2. 각 식탁의 상세 URL을 `curl -sSL -w %{url_effective}`로 **최종 목적지**까지 확인 (홈/검색/카테고리 리다이렉트 = 실패)
3. 이미지: 상세페이지 JSON-LD `image` → `og:image` → 공식 상품 이미지 순으로 확보, HTTP 200·`image/*`·해상도(너비≥300)·중복 없음 검증
4. 매칭 의자: 같은 브랜드/판매처의 실존 4인용 의자 이름 + 개당가격 확보 (URL/이미지 검증은 생략)
5. `setTotal = price + chairPrice*4` 계산 후 200만 기준으로 status/badge 결정
6. `node --check`(추출 스크립트) + `git diff --check`로 검증
7. 브랜드별로 커밋 후 `git push origin main`, 원격 반영 확인 후 다음 브랜드

## 완료 조건

```text
TABLES.length >= 100
식탁 url 홈/검색/카테고리 폴백 = 0 (전부 실제 상세페이지, HTTP 200)
식탁 img 존재 = 100, 전부 HTTP 200 · image/* · 중복 없음
의자 chairName / chairPrice 존재 = 100
setTotal 계산 정확 (= price + chairPrice*4)
중복 상품명 = 0
nav.js · index.html 링크 반영
```
